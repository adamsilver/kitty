Menu demo

Start with a menu bar. Without JS voiceover will announce the button is part of a menu for example. When JS available enhance with the following - give each button tabindex-1 so that the menubar is operable with the keyboard but not in the normal tab sequence. Menus with lots of buttons are a pain as user has to step through them all to leave the menu. - on small screens, when no space, change the role to menu, instead of menubar, as this indicates it's normal popupable i.e. revealed with a user action, and hidden otherwise. - also on small screens, create a button with so that the user focuses on the button and will know it has a menu with the popup attribute. Clicking it toggles the visibility of the menu on small screens. This button doesnt exist on big screens. - on big screens, continue to use menubar.
